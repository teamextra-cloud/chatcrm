# ChatCRM Coding Rules

When generating code follow these rules.

---

# Rule 1

Do not change database schema without explanation.

---

# Rule 2

All code must be copy-paste ready.

Do not output partial snippets.

---

# Rule 3

Use React functional components only.

---

# Rule 4

Use async/await.

Do not use callbacks.

---

# Rule 5

Always support both platforms:

LINE

Facebook Messenger

---

# Rule 6

When displaying customers list show:

avatar

name

platform icon

last message

---

# Rule 7

When displaying file messages show:

file icon

file name

file size

download link

---

# Rule 8

Never remove existing features when modifying code.

Only extend functionality.

---

# Rule 9

Frontend must update in realtime when new message arrives.

Use Supabase realtime events.

---

# Rule 10

Keep code modular.

Do not write very large components.
